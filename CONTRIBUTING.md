# Contributing to Noesis

Thank you for your interest in contributing to Noesis.

This repository is public so that people can review the work, open issues, suggest improvements, and propose contributions. Public visibility does not mean the project is open source unless the repository has a separate open-source license file that expressly says so.

## Project Status

Noesis is a governed project. The project owner maintains final authority over:

- roadmap
- canonical schemas
- taxonomy structure
- governance rules
- compatibility claims
- licensing
- releases
- repository organization
- use of project names, logos, and marks

Contributions are welcome, but contribution does not create ownership, control rights, maintainer status, or authority over project direction.

## Ways to Contribute

You may contribute by:

- opening issues
- reporting bugs
- proposing taxonomy additions
- suggesting documentation improvements
- submitting pull requests
- proposing compatibility mappings
- suggesting governance improvements
- identifying unclear or inconsistent project structure

Please keep contributions scoped, auditable, and tied to a clear issue or purpose where possible.

## Contribution Requirements

By submitting an issue, pull request, discussion comment, documentation edit, code contribution, schema, taxonomy entry, mapping, diagram, governance proposal, or other material, you represent that:

1. You have the right to submit the contribution.
2. Your contribution is your original work, or you have sufficient rights to submit it.
3. Your contribution does not contain confidential, privileged, proprietary, restricted, or third-party controlled material unless clearly authorized.
4. Your contribution does not knowingly violate another person’s copyright, trademark, patent, trade secret, privacy right, contractual obligation, or other legal right.
5. Your contribution may be reviewed, modified, rejected, accepted, reorganized, or used in a different form than submitted.

Contributions are voluntary.

By contributing, you understand that you are not entitled to compensation, royalties, ownership, equity, employment, contractor status, partnership status, attribution beyond ordinary project records, or control over project decisions unless separately agreed in writing.

## Review and Acceptance

The project owner or designated maintainers may accept, reject, modify, edit, reformat, combine, split, defer, or close contributions at their discretion.

A contribution is not accepted merely because it has been submitted, discussed, reviewed, or referenced.

Only merged pull requests or expressly approved materials should be treated as accepted project content.

## Canonical Authority

Do not create or modify canonical project structures unless specifically authorized.

This includes:

- canonical schema IDs
- taxonomy IDs
- governance IDs
- compatibility claims
- route authority
- controlled vocabulary
- official mappings
- official release labels
- license terms
- project branding
- logos and marks

Proposed changes to canonical structures should be submitted as proposals, not as assumed authority.

## Pull Request Guidelines

Pull requests should include:

- a short explanation of the change
- the reason for the change
- affected files
- any compatibility impact
- any governance impact
- any open questions
- any known risks or limitations

For larger changes, open an issue first.

## Issue Guidelines

Good issues should include:

- what problem you found
- where it appears
- why it matters
- proposed fix, if any
- screenshots, logs, examples, or citations where useful

Please avoid vague issues such as “fix this” or “improve taxonomy” without enough detail to evaluate the request.

## Compatibility Claims

Do not claim that a fork, plugin, adapter, taxonomy branch, or external project is officially compatible with Noesis, Logos, or related projects unless that compatibility has been approved by the project owner.

You may say that something is “proposed for compatibility review” or “designed to interoperate with” the project if that statement is accurate and not misleading.

## Names, Logos, and Marks

Project names, logos, marks, visual identity, and branding assets are controlled by the project owner.

You may not use Noesis, Logos, or related names or marks to imply endorsement, official status, partnership, certification, or compatibility without written permission.

## Sensitive or Restricted Material

Do not submit:

- confidential information
- client data
- privileged material
- private personal information
- trade secrets
- proprietary third-party content
- copyrighted material you do not have permission to contribute
- security vulnerabilities without following responsible disclosure practices

If you are unsure whether something is safe to contribute, do not include it.

## Security Issues

Please do not disclose security vulnerabilities publicly through issues or discussions.

Instead, use the process listed in `SECURITY.md` if available.

## Questions

If you are unsure whether a contribution is appropriate, open an issue describing the proposed contribution before submitting a pull request.
