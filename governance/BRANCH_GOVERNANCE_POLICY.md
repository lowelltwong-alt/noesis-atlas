# Branch Governance Policy

Each branch must define:

- branch ID;
- scope;
- branch head status;
- recognized authority sources;
- source tiers;
- internal governance logic;
- human/AI-in-loop stance;
- required reviewers;
- sub-branch split rules;
- branch-specific audit triggers;
- Logos crosswalk status.

## Branch status values

- `stub`
- `research_candidate`
- `community_proposed`
- `branch_reviewed`
- `curated`
- `logos_reviewed`
- `restricted`

## Split rule

If a tradition's sub-branch produces different answers on authority, worship, law/practice, metaphysics, personhood, moral governance, or AI permissibility, create a sub-branch rather than one generic profile.
