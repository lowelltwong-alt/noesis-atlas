# Governance Overview

Noesis Atlas uses a **governance-of-governances** model.

## Layer 1 — Noesis constitutional governance

Noesis decides what can enter the system, what can be promoted, what must be audited, what can be exported, and what is allowed to connect to Logos.

## Layer 2 — Branch-native governance

Each branch defines its own internal authority structure:

- Catholic: magisterium, Scripture, Tradition, councils, catechism.
- Sunni Islam: Qur’an, Sunnah, ijmāʿ, qiyās, madhhab-specific juristic reasoning.
- Orthodox Judaism: Torah, Mishnah/Talmud, codes, responsa, minhag/community custom.
- Vedanta: prasthāna-traya and school-specific bhāṣyas/commentary lineages.
- Secular naturalism: public reason, law, dignity, empirical evidence, institutional legitimacy.
- AI religion branches: internal authority may be AI/truth-test based, but Noesis blocks AI self-canonization.

## Layer 3 — Logos compatibility governance

Logos may consume only approved exports. Noesis classification does not define Logos doctrine.

## Branch-head model

Each religion/tradition should eventually have:

- a branch head;
- at least two branch reviewers from distinct perspectives where possible;
- sub-builders for sects, schools, denominations, rites, madhhabs, lineages, or regional practices;
- a conflict/escalation path;
- a public uncertainty register for unresolved branch disputes.

## Audit requirement

All promotions, demotions, removals, crosswalk exports, and governance changes must create an audit log event.
