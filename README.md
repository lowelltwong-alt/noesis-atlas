# Noesis Atlas

**Noesis Atlas** is a source-available, controlled-use worldview, philosophy, religion, and formal-systems taxonomy control plane.

It is designed to work with **Logos** without becoming Logos. Noesis classifies worldview structures, authority profiles, truth modes, branch governance, source roots, derivatives, provenance, and crosswalks. Logos consumes only reviewed, Logos-approved exports through a governed compatibility adapter.

## Core rule

> Noesis governs the system. Each branch governs meaning inside its own worldview. Logos, Judaism, Islam, secular naturalism, AI religion, and other routes do not share one permission model.

## What this repo owns

- Branch profile cards and branch governance profiles.
- Authority/source/root metadata.
- Contribution workflow for branch heads and sub-builders.
- Trust zones, lifecycle states, promotion/demotion/removal paths.
- Audit logs, provenance metadata, truth-integrity metadata, and derivative lineage.
- Logos compatibility contracts and safe exports.

## What this repo does not own

- It does not define Christian doctrine for Logos.
- It does not declare one universal religion/philosophy as true.
- It does not let AI-generated content become canonical without review.
- It does not permit branch-internal claims to override Noesis constitutional governance.

## MVP status

This starter repo is intentionally skeletal. Branch content is mostly placeholder material. The point is to create a safe structure for future expert contribution.


## Expanded branch trees and coherence layer

This expanded scaffold includes visible route trees for major branches:

- Christianity
- Judaism
- Islam
- Hindu traditions / Vedanta
- Buddhism
- Secular naturalism
- AI religion
- Math and formal systems

The major governance rule is **coherence**. Every branch must be graded for:

1. internal coherence inside its own authority structure; and
2. external coherence across Noesis parent/sibling branches, route contracts, provenance, lifecycle state, and Logos compatibility.

Start with:

- `docs/BRANCH_TREE_ATLAS.md`
- `governance/COHERENCE_POLICY.md`
- `docs/COHERENCE_SCORING_MODEL.md`
- `registry/route-tree-registry.json`
- `examples/coherence-reports/`
